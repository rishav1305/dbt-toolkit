# dbt Artifacts Reference

dbt generates JSON artifacts during execution that contain project metadata, execution results, and data catalog information.

## manifest.json

Generated by: `dbt parse`, `dbt compile`, `dbt run`, `dbt build`, `dbt docs generate`

The manifest is the complete project DAG with all metadata.

### Top-level structure

```json
{
  "metadata": { ... },
  "nodes": { ... },
  "sources": { ... },
  "exposures": { ... },
  "parent_map": { ... },
  "child_map": { ... },
  "macros": { ... },
  "docs": { ... },
  "selectors": { ... }
}
```

### metadata

Project and execution information.

```json
{
  "metadata": {
    "dbt_schema_version": "https://schemas.getdbt.com/dbt/manifest/v13.json",
    "dbt_version": "1.8.0",
    "generated_at": "2024-03-15T14:32:18.123456Z",
    "invocation_id": "3c9351f7-5d52-418b-9d5c-8d9b3f7c2e1a",
    "env": {},
    "adapter_type": "redshift",
    "adapter_version": "1.8.0"
  }
}
```

**Key fields:**
- `dbt_version` — dbt version used to generate manifest
- `generated_at` — ISO 8601 timestamp
- `invocation_id` — Unique run ID
- `adapter_type` — Database adapter (redshift, snowflake, postgres, etc.)

---

### nodes

All models, tests, snapshots, seeds, and analyses keyed by unique_id.

```json
{
  "nodes": {
    "model.my_project.my_model": {
      "unique_id": "model.my_project.my_model",
      "name": "my_model",
      "resource_type": "model",
      "package_name": "my_project",
      "path": "models/staging/stg_customers.sql",
      "original_file_path": "models/staging/stg_customers.sql",
      "depends_on": {
        "macros": [],
        "nodes": ["source.raw.customers"]
      },
      "compiled_code": "SELECT ...",
      "config": {
        "materialized": "table",
        "sort": ["date"],
        "dist": "customer_id",
        "tags": ["daily", "critical"]
      },
      "meta": {
        "owner": "analytics",
        "sla_hours": 24
      },
      "columns": {
        "customer_id": {
          "name": "customer_id",
          "description": "Customer ID",
          "data_tests": ["unique", "not_null"]
        }
      }
    },
    "test.my_project.not_null_customers_customer_id": {
      "unique_id": "test.my_project.not_null_customers_customer_id",
      "resource_type": "test",
      "attached_node": "model.my_project.my_model"
    }
  }
}
```

**Keys in each node:**
- `unique_id` — Globally unique identifier
- `resource_type` — model, test, snapshot, seed, analysis
- `path` — File path relative to project
- `depends_on` — List of upstream dependencies (nodes and macros)
- `compiled_code` — Final compiled SQL
- `config` — Model configuration
- `meta` — Custom metadata dict
- `columns` — Column definitions and tests

### sources

All defined sources keyed by unique_id.

```json
{
  "sources": {
    "source.raw.customers": {
      "unique_id": "source.raw.customers",
      "source_name": "raw",
      "name": "customers",
      "database": "analytics_db",
      "schema": "raw_schema",
      "identifier": "raw_customers_table",
      "columns": {
        "customer_id": {
          "name": "customer_id",
          "description": "Unique customer ID"
        }
      },
      "meta": {
        "owner": "data-team"
      },
      "freshness": {
        "warn_after": { "count": 24, "period": "hour" },
        "error_after": { "count": 48, "period": "hour" }
      }
    }
  }
}
```

---

### parent_map

Mapping from each node to its direct parents (upstream dependencies).

```json
{
  "parent_map": {
    "model.my_project.my_model": [
      "source.raw.customers",
      "model.my_project.base_customers"
    ],
    "model.my_project.final_customers": [
      "model.my_project.my_model"
    ]
  }
}
```

**Use case:** Build custom DAG visualizations or dependency tracers.

---

### child_map

Mapping from each node to its direct children (downstream dependents).

```json
{
  "child_map": {
    "source.raw.customers": [
      "model.my_project.my_model"
    ],
    "model.my_project.my_model": [
      "model.my_project.final_customers",
      "test.my_project.unique_my_model_id"
    ]
  }
}
```

---

### macros

All defined macros in the project and dependencies.

```json
{
  "macros": {
    "macro.my_project.my_macro": {
      "unique_id": "macro.my_project.my_macro",
      "name": "my_macro",
      "package_name": "my_project",
      "path": "macros/my_macro.sql",
      "arguments": [
        {
          "name": "input_arg",
          "type": "string",
          "description": "Input argument"
        }
      ]
    }
  }
}
```

---

### docs

Documentation blocks defined via `{% docs ... %}` in `.md` files.

```json
{
  "docs": {
    "my_project.my_doc": {
      "unique_id": "my_project.my_doc",
      "name": "my_doc",
      "block_contents": "This is documentation..."
    }
  }
}
```

## run_results.json

Generated by: `dbt run`, `dbt test`, `dbt build`, `dbt seed`, `dbt snapshot`, `dbt retry`

Execution results for every node in the run.

### Top-level structure

```json
{
  "metadata": { ... },
  "results": [ ... ],
  "elapsed_time": 123.45
}
```

### metadata

```json
{
  "metadata": {
    "dbt_schema_version": "https://schemas.getdbt.com/dbt/run-results/v6.json",
    "dbt_version": "1.8.0",
    "generated_at": "2024-03-15T14:32:45.789012Z",
    "invocation_id": "3c9351f7-5d52-418b-9d5c-8d9b3f7c2e1a",
    "env": {},
    "adapter_type": "redshift"
  }
}
```

---

### results[]

Array of per-node execution results.

```json
{
  "results": [
    {
      "unique_id": "model.my_project.my_model",
      "status": "success",
      "message": null,
      "execution_time": 12.34,
      "thread_id": "Thread-1",
      "adapter_response": {
        "code": "INSERT 0 15000",
        "_message": "INSERT 0 15000",
        "rows_affected": 15000
      },
      "compiled_code": "CREATE TABLE my_schema.my_model AS SELECT ...",
      "compiled": true,
      "timing": [
        {
          "name": "compile",
          "started_at": "2024-03-15T14:32:20.123456Z",
          "completed_at": "2024-03-15T14:32:25.123456Z"
        },
        {
          "name": "execute",
          "started_at": "2024-03-15T14:32:25.123456Z",
          "completed_at": "2024-03-15T14:32:45.789012Z"
        }
      ]
    },
    {
      "unique_id": "test.my_project.unique_my_model_id",
      "status": "fail",
      "message": "Assertion failed on test.my_project.unique_my_model_id",
      "execution_time": 2.10,
      "thread_id": "Thread-2",
      "adapter_response": {
        "code": "SELECT 0",
        "rows_affected": 2
      },
      "compiled_code": "SELECT customer_id, COUNT(*) as cnt FROM my_schema.my_model GROUP BY customer_id HAVING cnt > 1"
    }
  ]
}
```

**Key fields per result:**
- `unique_id` — Node identifier
- `status` — success, error, fail, skipped
- `message` — Error message if failed
- `execution_time` — Seconds to execute
- `compiled_code` — Generated SQL
- `adapter_response` — Row counts, bytes, etc. from database
- `timing` — Compile and execute phase timestamps

### Status values

- **success** — Node executed without errors
- **error** — Parse or execution error (SQL syntax, connection issues)
- **fail** — Test failed (assertion returned rows)
- **skipped** — Node was skipped (upstream failure, skip config, etc.)

## catalog.json

Generated by: `dbt docs generate`

Data catalog with column types, row counts, and table statistics.

### Structure

```json
{
  "metadata": {
    "dbt_schema_version": "...",
    "dbt_version": "1.8.0",
    "generated_at": "2024-03-15T14:32:45.789012Z",
    "invocation_id": "..."
  },
  "nodes": {
    "model.my_project.my_model": {
      "metadata": {
        "schema": "my_schema",
        "name": "my_model",
        "database": "analytics_db",
        "comment": "My model description",
        "owner": "analytics"
      },
      "columns": {
        "customer_id": {
          "name": "customer_id",
          "index": 1,
          "type": "integer",
          "comment": "Unique customer ID"
        },
        "created_at": {
          "name": "created_at",
          "index": 2,
          "type": "timestamp without time zone",
          "comment": "Creation timestamp"
        }
      },
      "stats": {
        "rows.estimate": 15000,
        "bytes.estimate": 1048576,
        "bytes_used": 1048576,
        "encoded": false,
        "ratio": 0.0
      }
    }

      "metadata": {
        "schema": "raw_schema",
        "name": "raw_customers_table",
        "database": "analytics_db"
```

**Use case:** Auto-generate data dictionaries, populate metadata registries.

## sources.json

Generated by: `dbt source freshness`

Freshness check results for all sources.

### Structure

```json
{
  "metadata": { ... },
  "results": [
    {
      "unique_id": "source.raw.customers",
      "status": "pass",
      "freshness_threshold": {
        "warn_after": { "count": 24, "period": "hour" },
        "error_after": { "count": 48, "period": "hour" },
        "warn_after_seconds": 86400,
        "error_after_seconds": 172800
      },
      "age": {
        "amount": 12,
        "unit": "hour"
      },
      "max_loaded_at": "2024-03-15T02:32:45Z",
      "max_loaded_at_time_ago_in_s": 43200
    },
    {
      "unique_id": "source.raw.transactions",
      "status": "warn",
      "max_loaded_at": "2024-03-14T14:32:45Z",
      "max_loaded_at_time_ago_in_s": 86400
    },
    {
      "unique_id": "source.raw.events",
      "status": "error",
      "error": "Table not found",
      "max_loaded_at": null
    }
  ]
}
```

**Status values:**
- **pass** — Data is fresh (within `warn_after` threshold)
- **warn** — Data is stale (between `warn_after` and `error_after`)
- **error** — Data is very stale or table not accessible (past `error_after`)
- **runtime error** — Query execution failed

**Key fields:**
- `max_loaded_at` — ISO 8601 timestamp of most recent data
- `max_loaded_at_time_ago_in_s` — Seconds since most recent data
- `age.amount` and `age.unit` — Human-readable age

## Using Artifacts in CI/CD

### Persist manifest for state tracking

```bash
dbt run --state previous_state/
dbt test --select state:modified --state previous_state/
```

Saves current manifest to `target/manifest.json` for comparison in next run.

---

### Parse run_results.json for success/failure

```bash
jq '.results[] | select(.status == "fail") | .unique_id' run_results.json
```

List all failed tests.

---

### Generate test report from run_results.json

```bash
jq '.results[] | select(.resource_type == "test") | {name: .unique_id, status: .status}' run_results.json
```

Export test results to a report format.

---

### Extract compilation errors from run_results.json

```bash
jq '.results[] | select(.status == "error") | {node: .unique_id, message: .message}' run_results.json
```

## Common Artifact Queries

### Find all tests for a model

```python
import json

with open('target/manifest.json') as f:
    manifest = json.load(f)

model_id = 'model.my_project.my_model'
tests = [
    node_id for node_id, node in manifest['nodes'].items()
    if node.get('resource_type') == 'test' and
       node.get('attached_node') == model_id
]
```

---

### Check if model has failures in run_results

```python
import json

with open('target/run_results.json') as f:
    results = json.load(f)

failed_nodes = [
    result['unique_id'] for result in results['results']
    if result['status'] in ['error', 'fail']
]
```

---

### Calculate total execution time by resource type

```python
import json
from collections import defaultdict

with open('target/run_results.json') as f:
    results = json.load(f)

with open('target/manifest.json') as f:
    manifest = json.load(f)

times_by_type = defaultdict(float)
for result in results['results']:
    resource_type = manifest['nodes'][result['unique_id']]['resource_type']
    times_by_type[resource_type] += result['execution_time']

for rtype, total_time in sorted(times_by_type.items()):
    print(f"{rtype}: {total_time:.2f}s")
```

|----------|---|---|---|
| `manifest.json` | parse, compile, run, docs generate | Project structure + metadata | Nodes, sources, DAG, macros, docs |
